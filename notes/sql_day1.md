What is a table?
A structured representation of a real-world entity (like customers, accounts, or transactions), where data is organized in rows and columns so it can be queried, validated, and analyzed reliably.

What is a row?
One complete record representing a single instance of the entity the table describes.

What is a column?
One specific type of information (attribute) that describes the entity represented by the table

How is a table similar to an Excel sheet or bank report?
A table is similar to an Excel sheet or bank report because it stores structured data in rows and columns, where each row represents one customer or record, and each column represents a specific type of information (like customer ID, name, or balance).

If you had a table called customers, what business questions could be asked from it?
What is the NPA ratio of customers?
Which customers maintain balances below the minimum required balance?
Which customers have frequent cheque returns?
Which customers have approved overdraft (POD) limits?
Which customers have excess balances outstanding for more than 90 days?
